## 🚀 1단계: 핵심 사용성 개선 (우선순위 최상)

### 1.1 홈 위젯 고도화

**현재**: 오늘/이번 달 통계만 표시
**제안**:
- 구성 가능한 위젯 설정 (표시할 카테고리 선택)
- 실시간 타이머 상태 표시 (측정 중일 때 경과 시간)
- 클릭 시 앱의 특정 탭으로 이동 (딥링크)

**예상 작업량**: 2~3일
**난이도**: 중

### 1.2 앱 바로가기 (App Shortcuts)

**제안**:
- 홈 화면 롱클릭 → "빠른 기록", "타이머 시작", "통계 보기" 바로가기
- `ShortcutManager` + `ShortcutInfoCompat` 사용

**예상 작업량**: 0.5일
**난이도**: 하

### 1.3 알림 채널 통합 및 스트릭 추적

**현재**: 타이머 알림 + 목표 리마인더 각각 별도 채널
**제안**:
- **연속 달성 스트릭** — 카테고리별 연속 기록일 수 계산, 7일/30일 달성 시 축하 알림
- **저녁 리마인더** — 설정한 시간에 오늘 미기록 카테고리 알림
- **목표 임박 알림** — 목표 기한 1일 전 푸시
- **WorkManager**를 사용하여 백그라운드에서 주기적 체크

**예상 작업량**: 3~4일
**난이도**: 중

### 1.4 기록 편집 UX 개선

**현재**: 수정 다이얼로그에서 카테고리/시간/노트 수정 가능
**제안**:
- **스와이프 to 수정/삭제** — `SwipeToDismiss`로 편집/삭제 액션
- **드래그 앤 드롭** — 항목 순서 변경
- **Undo Snackbar** — 삭제 시 "실행 취소" 스낵바 3초 표시
- **인라인 편집** — 항목 탭 한 번으로 노트 바로 수정

**예상 작업량**: 4~5일
**난이도**: 중상

## 📊 2단계: 데이터 분석 및 시각화 강화

### 2.1 상세 통계 페이지

**현재**: donut 차트 + 카테고리별 합계
**제안**:
- **시계열 라인 차트** — 일별/주별 시간 추이 (Canvas 또는 `Vico` 차트 라이브러리)

- **PDF 리포트** — `Android PdfDocument`로 월간 리포트 PDF 생성 및 공유

**예상 작업량**: 5~7일
**난이도**: 상

### 2.2 목표 대시보드

**현재**: 목표 카드 리스트 + 진행률 바
**제안**:

- **목표 자동 생성** — 지난 3개월 평균 기록을 기반으로 다음 달 목표 제안
- **스트릭 위젯** — 연속 달성일 수 홈 화면 표시

**예상 작업량**: 4~6일
**난이도**: 중상

## 🔐 3단계: Firebase 및 보안 강화

### 3.1 Firebase Security Rules

**현재**: Firestore 보안 규칙 미적용 (개발 모드)
**제안**:
```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /groups/{groupId} {
      allow read, write: if request.auth != null 
        && request.auth.uid in resource.data.members;
    }
    match /groups/{groupId}/entries/{entry} {
      allow read, write: if request.auth != null 
        && request.auth.uid in get(/databases/$(database)/documents/groups/$(groupId)).data.members;
    }
  }
}
```

**예상 작업량**: 1일
**난이도**: 하

### 3.2 FCM 푸시 알림

**현재**: Firebase는 연동되어 있으나 FCM 미사용
**제안**:
- 그룹 멤버가 새 항목 등록 시 다른 멤버들에게 Push Notification
- FCM 토큰 관리 및 Cloud Functions 연동
- `FirebaseMessagingService` 상속 구현

**예상 작업량**: 3~4일
**난이도**: 중

### 3.3 오프라인 동기화 개선

**현재**: Firestore offline persistence 미사용
**제안**:
- `FirebaseFirestore.getInstance().firestoreSettings = FirebaseFirestoreSettings.Builder().setPersistenceEnabled(true).build()`
- 오프라인에서 작성한 항목 큐잉 및 온라인 복구 시 자동 업로드
- 동기화 상태 아이콘 (⏳ ✅ ❌) 표시

**예상 작업량**: 2~3일
**난이도**: 중

### 3.4 Crashlytics 연동

**제안**:
- `FirebaseCrashlytics` 초기화
- `UncaughtExceptionHandler`로 크래시 로그 수집
- 비정상 종료 시 다음 실행 때 팝업으로 안내

**예상 작업량**: 1일
**난이도**: 하

## ⌚ 4단계: Wear OS 연동 심화

### 4.1 Wear OS 독립 실행 앱

**현재**: `wear` 모듈에 서비스만 존재
**제안**:
- Wear OS 전용 Compose UI (`WearApp`, `Scaffold`, `ScalingLazyColumn`)
- 음성 입력 → 메모 자동 기록 ( `SpeechRecognizer` + `ActivityResultContracts.CreateDocument` )
- Wear에서 기록한 항목 → Phone으로 자동 동기화 ( `MessageClient` / `DataClient` )
- Wear OS 타일(Tile) — 오늘 활동 요약 한눈에
- 컴플리케이션(Complication) — 시계 페이스에 오늘 기록 시간 표시

**예상 작업량**: 7~10일
**난이도**: 상

### Dependency Injection
```kotlin
// Hilt 도입 시 (선택사항)
@HiltAndroidApp
class PlannerApp : Application()

@HiltViewModel
class PlannerViewModel @Inject constructor(
    private val repository: Repository,
    @ApplicationContext private val context: Context

```

### Compose Migration Path
```kotlin
// Compose BOM → Kotlin 2.0 Compose Compiler
// build.gradle.kts
android {
    composeOptions {
        kotlinCompilerExtensionVersion = "2.0.0"  // Kotlin 2.0 대응
    }
}
// 또는 Kotlin 2.0+ Compose Compiler Gradle Plugin
plugins {
    id("org.jetbrains.kotlin.plugin.compose") version "2.0.0"
}
```
